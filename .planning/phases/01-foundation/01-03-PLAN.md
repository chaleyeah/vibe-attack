<context>
Phase: 01 (foundation) — Milestone: M001: Migration

Implement the two real-time subsystems that form the PTT pipeline: CPAL audio capture
(with warm stream + AtomicBool gate) and evdev PTT scanner (device enumeration + blocking
event loop). Both subsystems share a single `Arc<AtomicBool>` (ptt_active) — the PTT
thread writes it; the CPAL callback reads it.

Purpose: These two systems form the input side of the entire daemon. Subsequent plans
wire them together with the injection thread and daemon loop.

Output: `src/audio/mod.rs` (CPAL stream + HeapRb gate), `src/input/ptt.rs` (evdev
enumerate + PTT thread). Both compile and have unit tests.
</context>

<must_haves>
- "CPAL stream opens on the default input device and stays warm (not torn down between PTT presses)"
- "The negotiated StreamConfig (channels, sample_rate) is logged at INFO on startup"
- "While ptt_active is true, f32 samples are pushed to the HeapRb producer inside the callback"
- "While ptt_active is false, samples are silently discarded (no allocation, no blocking in callback)"
- "PTT thread scans all /dev/input/event* nodes and finds the first device reporting the configured key"
- "If no /dev/input/event* node is readable, daemon fails with actionable error mentioning 'input' group"
- "PTT key press sets ptt_active to true; release sets it to false; both logged at TRACE only"
- "cargo test --lib exits 0 with audio and PTT unit tests passing"
</must_haves>
