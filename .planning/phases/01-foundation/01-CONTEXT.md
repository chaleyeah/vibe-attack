# Phase 1: Foundation - Context

**Gathered:** 2026-04-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Deliver a headless CLI daemon that can:
1. Capture microphone audio via CPAL (push-to-talk gated)
2. Inject arbitrary key sequences into a Wayland session via a uinput virtual keyboard
3. Detect and hold a PTT key via evdev grab from any input device — working correctly even when a fullscreen Proton/Steam game holds compositor focus
4. Start with a minimal YAML config and fail-fast with actionable errors on missing privileges

New capabilities (VAD, STT, phrase matching, GUI, pack system) are out of scope for this phase.

</domain>

<decisions>
## Implementation Decisions

### Audio Capture

- **D-01:** Use **CPAL** as the audio capture backend (cross-driver: works via ALSA, PulseAudio, PipeWire depending on the system)
- **D-02:** Default to the **system default input device** — no explicit device path required in Phase 1 config
- **D-03:** Capture format is **mono 16 kHz f32** (resample at the source; no deferred resample cost)
- **D-04:** PTT capture semantics — **keep the stream running warm** at all times; **gate samples** to the pipeline only while PTT is held (option b). Stream start/stop is NOT tied to PTT press.

### Input Injection

- **D-05:** uinput virtual device presents as a **keyboard-only** device (no mouse axes). Mouse macros can be added in a later phase with a new device node if needed.
- **D-06:** Key timing uses **global defaults** (dwell duration, inter-key gap) with **per-key override** in the macro definition. Defaults are read from config at startup.
- **D-07:** Injection executes on its **own dedicated OS thread** (blocking, not on the Tokio executor) — consistent with the project-level rule that latency-sensitive work stays off the async runtime.
- **D-08:** Key-event logging is **silent by default**; every event is emitted at `TRACE`/`DEBUG` level and only visible when `--verbose` is passed on the CLI.

### PTT Behavior

- **D-09:** The daemon scans **all `/dev/input/event*` nodes** and grabs the first device that reports the configured PTT keycode — no explicit device path required in config.
- **D-10:** PTT grab is **non-exclusive** — other applications (including the game) continue to receive the PTT key event. `EVIOCGRAB` is not used.
- **D-11:** If the PTT device cannot be opened or grabbed, the daemon **fails immediately with a clear error** (no fallback, no silent degradation).
- **D-12:** PTT press/release events are **suppressed unless `--verbose`** is passed — consistent with injection logging policy.

### Daemon + Config

- **D-13:** Config file format is **YAML** (`config.yaml`)
- **D-14:** Config file location follows **XDG**: `$XDG_CONFIG_HOME/hd-linux-voice/config.yaml` (falls back to `~/.config/hd-linux-voice/config.yaml` when `XDG_CONFIG_HOME` is unset)
- **D-15:** `/dev/uinput` startup failure prints an actionable error in the form:
  ```
  Error: cannot open /dev/uinput — permission denied.
  Fix: sudo modprobe uinput && sudo usermod -aG input $USER
  Then log out and back in, or run: newgrp input
  Docs: https://github.com/<org>/hd-linux-voice/blob/main/docs/uinput-setup.md
  ```
  (Exact org/path TBD; planner should use the real repo URL.)
- **D-16:** `LICENSES.md` is generated by **`cargo-about`** (auto, stays current with dep tree). The AGPL-3.0 license file already exists in the repo root and should be **excluded from the inventory** (no need to re-list the project's own license).

<specifics>
## Specific Ideas

- CPAL chosen specifically for its cross-driver portability (works on ALSA, PulseAudio, PipeWire without recompile)
- Mono 16 kHz f32 is the exact format whisper.cpp expects — capturing in this format avoids a resample stage in Phase 2
- "Gate samples" model (warm stream, gate to pipeline) avoids the latency hit of stream teardown/reinit on every PTT press

</specifics>
