<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| curl pipe to sh | rustup installer fetched from internet and executed as user |
| crates.io network | Dependency resolution fetches from crates.io; Cargo.lock pins hashes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Spoofing | rustup installer (curl pipe) | accept | Official Rust foundation URL (sh.rustup.rs) over TLS 1.2+; standard ecosystem practice |
| T-01-01-02 | Tampering | Cargo.toml dependency versions | mitigate | Cargo.lock pins all resolved crate hashes after first `cargo check`; commit Cargo.lock to VCS |
| T-01-01-03 | Elevation of Privilege | serde_yaml CVE (old crate) | mitigate | Cargo.toml declares `serde_yaml_ng` (MIT, no CVE); grep check ensures `serde_yaml ` (with space) never appears |
| T-01-01-04 | Elevation of Privilege | GUI surface crates | mitigate | about.toml `accepted` list omits GUI-related licenses; acceptance_criteria grep confirms absence of winit/xcb/wayland-client |
</threat_model>
