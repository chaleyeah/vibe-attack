  <action>
    Implement two new fields in the stable stdout JSONL “utterance” event, while preserving the stdout/stderr split (D-19..D-22):

    1) Add an **explicit end-to-end duration field** on the utterance JSON:
       - Field name: `e2e_ms`
       - Definition (per verifier recommendation): `e2e_ms = output_done_ms - vad_done_ms`
       - Compute it at emission time (in `JsonlWriter::write_utterance` when `output_done_ms` is known), and include it in the `JsonlEvent::Utterance` struct.
       - Use `saturating_sub` for safety, but in correct wiring this should be non-negative.

    2) Add a **measured VAD compute duration field** that is separate from utterance timeline/length:
       - Field name: `vad_ms`
       - Definition: sum of per-frame “VAD scoring + segmentation decision” compute time for frames belonging to the utterance.
       - Measurement strategy:
         - Time the Silero forward pass (or the full `push_frame_silero` path if easiest/cleanest) using `std::time::Instant`.
         - Accumulate compute ms only for the active utterance (reset when an utterance starts; attach total when the utterance ends).
         - Store the accumulated value inside `UtteranceTimings` (add `vad_ms: Option<u64>` or equivalent) so it survives the STT thread and reaches the output thread unchanged.
       - The goal is “VAD stage cost” budgeting (≤ 50ms) per Phase 2 verification, not utterance length.

    Constraints:
    - **Stdout remains JSONL-only.** Any instrumentation stays on stderr (tracing) — do not add println/debug output.
    - Keep changes localized to the existing Phase 2 patterns (see 02-02/02-03 summaries): timing is carried through `UtteranceTimings` and emitted as u64 ms fields.
  </action>
  <verify>
    <automated>cargo test -q</automated>
  </verify>

    - VAD compute time is measured and propagated through to the stdout JSONL utterance event as `vad_ms`.
    - No new stdout writes are introduced outside `JsonlWriter`.
  </acceptance_criteria>
