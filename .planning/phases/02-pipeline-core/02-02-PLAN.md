    Create `src/vad/mod.rs` implementing VAD scoring + utterance segmentation on an OS thread (do NOT touch the CPAL callback).

    Required behaviors (locked decisions):
    - D-01/D-09: Treat pipeline “frame clock” as 20ms frames (320 samples at 16kHz) and produce a VAD decision per 20ms frame.
    - Research constraint: Silero expects 512-sample windows at 16kHz; implement a sliding window buffer of 512 samples advanced every 320 samples while still emitting 20ms-resolution scores.
    - D-10: Hysteresis: start threshold > stop threshold.
    - D-12: Minimum speech (~100ms) before committing to an utterance.
    - D-11: End-of-speech requires ~400ms silence.
    - D-06/D-07: Pre-roll and tail buffers (~100–200ms each) are included in utterance audio.
    - D-08: Hard cap utterance length ~10s; force cut/flush and log warning (stderr) when exceeded.
    - D-04: When PTT is held, still run VAD to trim silence; PTT gates capture but not segmentation logic.
    - D-05: Poll + short sleep when insufficient samples; no busy spin.

    Output contract for downstream STT:
    - Define a small “utterance job” struct that contains:
      - utterance id
      - audio buffer (Vec<f32> built on the VAD thread, NOT in audio callback)
      - stage timing markers from `pipeline/timing`
      - metadata (start/end frame indices, durations) needed for latency reporting

    Update `tests/drop_oldest_queue.rs` so it tests the exact “drop-oldest” algorithm the STT queue will use (per D-03), independent of VAD itself.
