# Plan 04-01: Control Channel Foundation

**Wave**: 1
**Depends on**: Phase 3 completion
**Autonomous**: true
**Files modified**: `Cargo.toml`, `src/control/mod.rs` [NEW], `src/control/protocol.rs` [NEW], `src/main.rs`

## Goal
Implement the Unix Domain Socket (UDS) control channel foundation and protocol. This allows the daemon to receive commands (like profile switching) from the CLI at runtime.

## Tasks

<task name="dependencies" read_first="Cargo.toml">
<action>
Add the following dependencies to `Cargo.toml`:
- `zip = "0.6"`
- `ratatui = "0.26"`
- `sha2 = "0.10"`
- `crossterm = "0.27"`
- `serde_json = "1.0"` (ensure it's present for the UDS protocol)
</action>
<acceptance_criteria>
`Cargo.toml` contains all requested crates.
`cargo check` passes.
</acceptance_criteria>
</task>

<task name="uds_protocol" read_first="src/config.rs">
<action>
Create `src/control/protocol.rs` with the following JSON-RPC-like structures:
- `ControlRequest` enum: `Ping`, `SwitchProfile { name: String }`, `TestMacro { name: String }`.
- `ControlResponse` enum: `Ok`, `Pong`, `Error { message: String }`.
Derive `Serialize` and `Deserialize` for all.
</action>
<acceptance_criteria>
`src/control/protocol.rs` exists and compiles.
Enums are exhaustive for current needs.
</acceptance_criteria>
</task>

<task name="uds_listener" read_first="src/main.rs">
<action>
Create `src/control/mod.rs` to host the `spawn_control_thread` function.
- It should use `tokio::net::UnixListener`.
- Socket path from `xdg::BaseDirectories::get_runtime_file("hd-linux-voice.sock")`.
- It should handle incoming connections in a loop, parsing JSON lines and (for now) just logging them.
- Wire it into `src/main.rs` (spawn it before the pipeline starts).
</action>
<acceptance_criteria>
Daemon starts and creates the `.sock` file in the runtime directory.
Logs show "Control channel listening on ..."
</acceptance_criteria>
</task>

## Verification
- Run the daemon and verify `ls -l /run/user/$UID/hd-linux-voice.sock` (or equivalent XDG path) shows the socket.
- Use `socat` or `nc -U` to send `{"cmd": "Ping"}` and verify it's received in the logs.
