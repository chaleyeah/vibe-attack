# Plan 04-06: TUI Edit & Live Test

**Wave**: 3
**Depends on**: 04-05-PLAN.md, 04-02-PLAN.md
**Autonomous**: true
**Files modified**: `src/tui/app.rs`, `src/tui/editor.rs` [NEW]

## Goal
Implement the macro editing view and the "Live Test" feature in the TUI.

## Tasks

<task name="edit_view" read_first="src/tui/app.rs">
<action>
Create `src/tui/editor.rs` to handle editing a single `MacroConfig`.
- Support editing `name`, `phrase`, `if_flag`, `set_flag`.
- A sub-view for editing the `keys` sequence (add/remove/reorder steps).
- A sub-view for dwell/gap overrides.
- Save changes back to `pack.yaml` on 'Ctrl+S'.
</action>
<acceptance_criteria>
Changes made in the TUI are persisted to the source `pack.yaml` file.
</acceptance_criteria>
</task>

<task name="live_test" read_first="src/control/client.rs">
<action>
Add a "Test" button (or 'T' key) in the TUI editor.
- When pressed, it sends a `TestMacro { name }` command to the daemon via the UDS client.
- The daemon should execute the macro immediately.
</action>
<acceptance_criteria>
Pressing 'T' in the editor triggers the macro in the running daemon.
This works even if the macro was just edited (but not yet switched to as the active profile).
</acceptance_criteria>
</task>

## Verification
- Edit a macro's key sequence in the TUI.
- Press 'T' and verify the daemon executes the NEW sequence (using the test command).
- Save and verify the file on disk.
