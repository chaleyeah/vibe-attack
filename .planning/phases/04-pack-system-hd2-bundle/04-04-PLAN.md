# Plan 04-04: Profile Registry & Hot-reload

**Wave**: 2
**Depends on**: 04-01-PLAN.md, 04-03-PLAN.md
**Autonomous**: true
**Files modified**: `src/pipeline/dispatcher.rs`, `src/control/mod.rs`, `src/main.rs`

## Goal
Enable the daemon to hot-reload its macro registry when a `SwitchProfile` command is received.

## Tasks

<task name="registry_update" read_first="src/pipeline/dispatcher.rs">
<action>
Update the `Dispatcher` (or its `MacroRegistry`) to support runtime updates.
- Wrap the registry in an `Arc<RwLock<...>>`.
- Add a method to swap the active macros with a new set from a `Pack`.
</action>
<acceptance_criteria>
The dispatcher can receive a new macro set without stopping its thread.
</acceptance_criteria>
</task>

<task name="hot_reload_handler" read_first="src/control/mod.rs">
<action>
Implement the handler for `SwitchProfile { name }` in the control channel thread.
- Locate the profile in `~/.config/hd-linux-voice/profiles/<name>/pack.yaml`.
- Load and flatten the pack.
- Update the dispatcher's registry.
- Reply with `Ok` if successful.
</action>
<acceptance_criteria>
Sending `switch_profile` via UDS causes the daemon to load the new macros.
The daemon logs the switch event.
</acceptance_criteria>
</task>

## Verification
- Load a profile A, speak a phrase from it.
- Send `switch_profile B`.
- Speak a phrase from profile B and verify it matches (and profile A's phrases no longer match).
