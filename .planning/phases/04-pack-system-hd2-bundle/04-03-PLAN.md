# Plan 04-03: Pack Management Logic

**Wave**: 2
**Depends on**: 04-01-PLAN.md, 04-02-PLAN.md
**Autonomous**: true
**Files modified**: `src/config.rs`, `src/pack/mod.rs` [NEW], `src/main.rs`

## Goal
Implement the core logic for `.hdpack` (ZIP) files, including importing, exporting, and loading the `pack.yaml` structure with category support.

## Tasks

<task name="pack_schema" read_first="src/config.rs">
<action>
Create `src/pack/mod.rs` and define the `Pack` and `Category` structs.
- `Pack` should contain `name`, `author`, and `categories: Vec<Category>`.
- `Category` contains `name` and `macros: Vec<MacroConfig>`.
Implement `Serialize` and `Deserialize` for these.
Add a `flatten()` method to `Pack` that returns a `Vec<MacroConfig>`.
</action>
<acceptance_criteria>
`Pack` struct correctly models the nested category structure.
`flatten()` produces a list of macros ready for the `MacroRegistry`.
</acceptance_criteria>
</task>

<task name="zip_logic" read_first="src/pack/mod.rs">
<action>
Implement `Pack::export(path: PathBuf)` using the `zip` crate.
- Write `pack.yaml` to the root of the ZIP.
- Copy all referenced sound files into a `sounds/` directory inside the ZIP.
Implement `Pack::import(path: PathBuf)`:
- Extract ZIP to `~/.config/hd-linux-voice/profiles/<pack_name>/`.
- Validate `pack.yaml` exists and is valid.
- Implement path traversal protection (verify no `..` in ZIP entries).
</action>
<acceptance_criteria>
Exporting a pack creates a valid `.hdpack` file.
Importing a pack correctly extracts files and rejects path traversal attempts.
</acceptance_criteria>
</task>

<task name="cli_import_export" read_first="src/main.rs">
<action>
Add `import <FILE>` and `export <NAME>` commands to the CLI.
</action>
<acceptance_criteria>
`hd-voice import mypack.hdpack` successfully installs the pack to the profiles directory.
</acceptance_criteria>
</task>

## Verification
- Create a dummy pack with one macro and a sound file.
- Export it, then import it under a different name.
- Verify the `pack.yaml` and sound file are correctly placed in the `profiles/` directory.
