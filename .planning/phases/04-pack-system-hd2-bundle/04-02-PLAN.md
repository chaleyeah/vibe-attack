# Plan 04-02: CLI Client & Switching

**Wave**: 1
**Depends on**: 04-01-PLAN.md
**Autonomous**: true
**Files modified**: `src/main.rs`, `src/control/client.rs` [NEW]

## Goal
Implement the CLI client side of the control channel, allowing users to send commands to the running daemon.

## Tasks

<task name="control_client" read_first="src/control/protocol.rs">
<action>
Implement `src/control/client.rs` with a `send_command(req: ControlRequest) -> Result<ControlResponse>` helper.
It should connect to the UDS socket, send the JSON-serialized request, and wait for a response.
</action>
<acceptance_criteria>
`send_command` correctly handles connection errors and timeout.
</acceptance_criteria>
</task>

<task name="cli_commands" read_first="src/main.rs">
<action>
Update the `clap` CLI in `src/main.rs` to add new subcommands:
- `ping`: Sends a `Ping` request to the daemon.
- `switch <NAME>`: Sends a `SwitchProfile` request.
- `test <NAME>`: Sends a `TestMacro` request.
Handle the responses and print them nicely to the terminal.
</action>
<acceptance_criteria>
`hd-voice ping` returns "Pong" (if daemon is running).
`hd-voice switch my-profile` sends the correct JSON.
</acceptance_criteria>
</task>

## Verification
- Start the daemon in one terminal.
- Run `hd-voice ping` in another and verify the response.
