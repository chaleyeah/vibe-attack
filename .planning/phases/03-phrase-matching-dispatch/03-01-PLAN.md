# Wave 1: Configuration & Fuzzy Matching Logic

This wave adds the necessary configuration structures and the core fuzzy matching engine using the `strsim` crate.

## Tasks

<task>
<read_first>
- Cargo.toml
- src/config.rs
</read_first>
<action>
1. Add `strsim = "0.11"` to `Cargo.toml`.
2. Update `SttConfig` in `src/config.rs` to include `confidence_threshold: f32` (default 0.8).
3. Update `MacroConfig` in `src/config.rs` to include:
   - `phrase: Option<String>`
   - `if_flag: Option<String>`
   - `set_flag: Option<String>`
   - `sound: Option<std::path::PathBuf>`
</action>
<acceptance_criteria>
- `cargo check` passes.
- `Config` struct successfully deserializes macros with the new fields.
</acceptance_criteria>
</task>

<task>
<read_first>
- src/config.rs
</read_first>
<action>
Create `src/pipeline/matcher.rs` and implement a `PhraseMatcher` that:
1. Normalizes input (lowercase, trim, strip punctuation).
2. Calculates similarity using `strsim::levenshtein`.
3. Returns the best match above the threshold.
</action>
<acceptance_criteria>
- Unit tests for `PhraseMatcher` cover:
  - Exact match.
  - Fuzzy match (e.g., "eagal" -> "eagle").
  - Threshold rejection.
  - Punctuation stripping.
</acceptance_criteria>
</task>
