# S05 Research: UI + Distribution

**Slice:** S05 — egui config window, system tray, first-run wizard, AppImage, AUR/PKGBUILD
**Researched:** 2026-04-25
**Confidence:** HIGH (existing codebase well-understood; egui/AppImage are standard Rust patterns)

---

## Summary

S05 must deliver **tests that prove the UI + Distribution contracts**, following the same scaffold pattern as S03 (phrase matching) and S04 (pack bundle). The slice is NOT building a fully functional egui GUI in the traditional sense — it is:

1. Creating a `src/ui/` module with a separate `hd-linux-voice-ui` binary backed by egui/eframe
2. Establishing a `FirstRunState` / `ConfigWindowState` module with unit-testable pure logic
3. Writing a `tests/ui_distribution.rs` integration test file proving those contracts hold
4. Adding packaging scaffolding (PKGBUILD template, AppImage build script) with tests verifying they are well-formed

The architectural constraint **UI-01** (daemon must remain headless) is already enforced by `tests/daemon_headless.rs` — the ldd check forbids `libwayland-client`, `libX11`, `libxcb`, `libgtk`, `libgdk` from the daemon binary. S05 must not break this invariant.

## Implementation Landscape

### What Already Exists

**`src/tui/`** — A working ratatui-based TUI (three files):
- `src/tui/mod.rs` — `run_editor()` entry point using crossterm + ratatui
- `src/tui/app.rs` — `App` struct with `AppMode` (Browser/Editor), `draw()`, `handle_key()`
- `src/tui/editor.rs` — `MacroEditor` struct for rendering a single `MacroConfig`

The TUI is already accessible via `hd-linux-voice edit` (CLI subcommand). It loads the first profile found in XDG profiles dir, renders a two-panel layout (categories + macros), and supports arrow-key navigation.

**`src/control/`** — Unix Domain Socket IPC (three files):
- `src/control/protocol.rs` — `ControlRequest` (Ping, SwitchProfile, TestMacro, Shutdown) + `ControlResponse` (Ok, Pong, Error)
- `src/control/client.rs` — `send_command()` via `$XDG_RUNTIME_DIR/hd-linux-voice/hd-linux-voice.sock`
- `src/control/mod.rs` — `spawn_control_listener()` Tokio task; handles `SwitchProfile` with ProfileManager persistence

**`src/error.rs`** — `DaemonError::UinputPermissionDenied` with actionable message linking to `docs/uinput-setup.md`

**`tests/daemon_headless.rs`** — Three existing tests:
- `daemon_binary_does_not_link_gui_libraries` — ldd check for forbidden GUI libs
- `daemon_exits_with_error_on_missing_config` — fail-fast behavior
- `uinput_permission_denied_message_links_to_docs` — D-15 error message check

**`Cargo.toml` — Current deps relevant to S05:**
- `ratatui = "0.26"`, `crossterm = "0.27"` — already in project for TUI
- **`egui` / `eframe` NOT present** — must be added as a new feature or separate crate feature

**`LICENSES.md`** — Generated by `cargo-about`; `about.toml` accepts MIT/Apache-2.0/LGPL etc. egui is MIT/Apache-2.0, so no license conflicts.

### What Needs to Be Built

**1. `src/ui/mod.rs` — UI state module (pure Rust, no GPU/window required for unit tests)**

A headless-testable state layer that the egui render loop drives:

```
src/ui/
  mod.rs       — pub re-exports
  first_run.rs — FirstRunState: detects missing config, guides model download
  config_app.rs — ConfigApp: profile picker, log view, mic level, macro list
```

Key design: `FirstRunState` and `ConfigApp` are plain structs with pure methods — they have **no egui imports**. The egui render functions are in a separate `src/ui/render.rs` (or feature-gated). This allows unit tests to call `FirstRunState::new()`, `is_setup_complete()`, `wizard_steps_remaining()` etc. without needing a display.

**2. `src/ui/first_run.rs` — First-run wizard state**

The wizard detects and guides through:
- Step 1: config file missing → emit path + copy command for `config.example.yaml`
- Step 2: `stt.enabled=true` but `model_path` missing or file absent → emit download command
- Step 3: `/dev/uinput` not accessible → emit `docs/uinput-setup.md` link + modprobe command
- Step 4: PTT key not found → guide to `evtest` + `--list-devices`

`FirstRunState` fields:
```rust
pub struct FirstRunState {
    pub config_exists: bool,
    pub model_path_valid: bool,
    pub uinput_accessible: bool,
    pub ptt_device_found: bool,
}

impl FirstRunState {
    pub fn detect() -> Self { ... }  // reads filesystem, not testable directly
    pub fn from_checks(config_exists: bool, model_path_valid: bool, uinput_accessible: bool, ptt_device_found: bool) -> Self
    pub fn is_setup_complete(&self) -> bool
    pub fn steps_remaining(&self) -> Vec<SetupStep>
    pub fn first_incomplete_step(&self) -> Option<SetupStep>
}
```

`SetupStep` enum: `CreateConfig`, `InstallModel`, `SetupUinput`, `ConfigurePtt`

This pure interface is fully testable without a display.

**3. `src/ui/config_app.rs` — Config window state**

```rust
pub struct ConfigApp {
    pub profiles: Vec<String>,
    pub active_profile: Option<String>,
    pub log_lines: Vec<String>,        // last N transcript events
    pub mic_level: f32,                // 0.0–1.0 RMS
}

impl ConfigApp {
    pub fn new() -> Result<Self>
    pub fn switch_profile(&mut self, name: &str) -> Result<()>  // calls send_command
    pub fn add_log_line(&mut self, line: String)
    pub fn profile_count(&self) -> usize
}
```

**4. `src/main.rs` addition: `hd-linux-voice-config` binary (or `config` subcommand)**

The egui window launches as a **separate process** (not linked into the daemon). Two approaches:
- Add `[[bin]] name = "hd-linux-voice-config"` in Cargo.toml
- Or keep it as a `hd-linux-voice config` subcommand that execs into egui

The daemon binary must NOT grow GUI deps (UI-01). The cleanest approach: a second `[[bin]]` entry that depends on `eframe` feature. Feature-gate: `[features] gui = ["eframe"]`.

**5. `packaging/` directory (new)**

```
packaging/
  PKGBUILD          — AUR/PKGBUILD template for Arch/CachyOS
  appimage/
    build.sh        — cargo build --release + linuxdeploy + appimagetool steps
    AppImageBuilder.yml  — optional: appimage-builder config
    hd-linux-voice.desktop  — .desktop file for AppImage
    hd-linux-voice.png      — icon (placeholder)
```

Tests for packaging files validate structure (PKGBUILD has required fields, .desktop file has required keys) — not runtime packaging.

**6. `tests/ui_distribution.rs` — test file**

Following the S04 pattern (hermetic, pure logic tests), this test file covers:

- `first_run_complete_when_all_checks_pass` — `from_checks(true,true,true,true).is_setup_complete() == true`
- `first_run_incomplete_when_config_missing` — steps_remaining contains CreateConfig
- `first_run_incomplete_when_model_missing` — steps_remaining contains InstallModel
- `first_run_incomplete_when_uinput_inaccessible` — steps_remaining contains SetupUinput
- `first_run_incomplete_when_ptt_missing` — steps_remaining contains ConfigurePtt
- `first_run_step_ordering_config_before_model` — wizard returns CreateConfig before InstallModel
- `first_run_all_steps_when_fresh_install` — `from_checks(false,false,false,false)` returns 4 steps
- `first_incomplete_step_returns_none_when_done` — `first_incomplete_step()` is None when complete
- `config_app_profile_count_reflects_profiles` — `ConfigApp { profiles: vec!["hd2"], .. }.profile_count() == 1`
- `config_app_add_log_line_grows_log` — add_log_line increments log_lines.len()
- `config_app_log_capped_at_max` — add_log_line doesn't grow past MAX_LOG_LINES
- `pkgbuild_file_exists_and_has_required_fields` — reads `packaging/PKGBUILD`, asserts pkgname/pkgver/url/license present
- `desktop_file_exists_and_has_required_keys` — reads `.desktop` file, asserts Name/Exec/Type present
- `appimage_build_script_is_executable` — `packaging/appimage/build.sh` has execute bit set
- `daemon_binary_feature_flag_separates_gui_from_daemon` — verify `gui` feature not in daemon default features
- `ui_module_is_not_pub_from_lib_without_gui_feature` (optional: compile-time gate test)

Estimated count: **15–18 tests** in `tests/ui_distribution.rs`.

## Critical Constraints

### UI-01: Daemon Must Remain Headless
`tests/daemon_headless.rs::daemon_binary_does_not_link_gui_libraries` enforces this via ldd. The S05 implementation **must not add egui/eframe to the default feature set** of the daemon binary. Use a separate `[[bin]]` or feature-gate `gui`.

### P-04: XWayland Focus Race
The config window, if opened as a daemon-linked subsystem, can minimize a fullscreen XWayland game. The correct architecture: daemon spawns the config process separately (or user launches `hd-linux-voice-config` manually). The daemon itself never maps a window.

### ORT + AppImage FUSE Mount Gotcha
`ort` crate (used by Silero-VAD) ships a bundled `libonnxruntime.so`. When running inside AppImage's FUSE mount, the dynamic linker path for this `.so` must be handled — either via `LD_LIBRARY_PATH` in the AppRun script or by compiling ORT statically. The AppImage `build.sh` must explicitly add the ORT `.so` to the AppDir and set `LD_LIBRARY_PATH` accordingly. This is the highest-risk packaging step.

### No Flatpak
The project correctly avoids Flatpak. PKGBUILD and AppImage are the two packaging targets. The `install.sh` companion to AppImage must run privileged setup (modprobe uinput, usermod -aG input) since AppImage cannot use post-install hooks.

### First-Run Model Download
The first-run wizard does NOT download models automatically (no silent downloads per research P-13). It emits the exact `curl` command the user should run to download `ggml-tiny.en.bin` to `~/.local/share/hd-linux-voice/models/whisper/`.

## Natural Seams (Task Decomposition Hints)

The work divides into four independent units:

**Unit A — `src/ui/first_run.rs` + tests (T01)**
- Pure struct + methods, no egui imports
- All `first_run_*` tests in `tests/ui_distribution.rs`
- Blocks nothing downstream; can be built and tested immediately

**Unit B — `src/ui/config_app.rs` + tests (T02)**
- Pure struct + methods; IPC calls go through `control::client::send_command` (already exists)
- All `config_app_*` tests
- Depends on A only for module setup

**Unit C — egui render layer + separate binary (T03)**
- `src/ui/render.rs` or `src/bin/hd-linux-voice-config.rs`
- Feature-gates `egui`/`eframe` behind `[features] gui = ["eframe"]`
- Adds `[[bin]]` entry, wires `FirstRunState` and `ConfigApp` to egui widgets
- The daemon ldd test continues to pass because daemon doesn't use `gui` feature

**Unit D — Packaging scaffolding + tests (T04)**
- Creates `packaging/PKGBUILD`, `packaging/appimage/build.sh`, `.desktop` file
- `pkgbuild_*` and `desktop_*` and `appimage_*` tests
- Independent of A/B/C — can be done in parallel or last

## What to Build First

**Start with Unit A (`FirstRunState`)** — it's pure logic, no deps, and its tests establish the slice's testability immediately. The `from_checks()` constructor (not `detect()`) is the unit-testable entry point.

**Then Unit B (`ConfigApp`)** — also pure, uses existing `send_command` for non-test paths.

**Then Unit C** — wire egui render; this is where the build complexity lives (feature gates, new binary entry). The key risk: Cargo.toml feature gating must not accidentally pull egui into the daemon's default dep tree.

**Then Unit D** — packaging files are the safest to add last; the tests are file-existence + content checks.

## Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| egui bleeds into daemon binary | HIGH | Feature-gate `gui`; verify ldd test still passes |
| ORT `.so` path broken in AppImage | MEDIUM | Document FUSE workaround in `build.sh`; test is existence check only |
| egui Wayland backend requires compositor | LOW | Tests use pure state structs, no display needed |
| PKGBUILD version drift | LOW | Tests check required fields exist, not version values |
